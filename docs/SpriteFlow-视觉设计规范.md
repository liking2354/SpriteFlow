# SpriteFlow 视觉设计规范

> 现代科技风设计语言 · 用于平台展示层（编辑器、素材库、控制台）

设计基调：**深色专业工具 + 精准霓虹点缀 + 玻璃态层次 + 数据网格**
参考坐标：Linear 的克制、Vercel 的高级感、ComfyUI 的节点表达力。

---

## 1. 设计原则

| 原则 | 含义 |
|------|------|
| **暗色优先** | 节点编辑/长时间创作场景，暗色护眼且凸显内容（精灵图、预览） |
| **内容发光，UI 隐退** | 界面是黑灰底，颜色只用在数据、状态、连线上——让用户视线聚焦素材本身 |
| **霓虹做点缀，不做主色** | 蓝/紫/青只用于强调（连线、激活、状态灯），大面积仍是中性灰 |
| **层次靠明度与玻璃态** | 不用重边框，用背景明度递进 + 轻微 blur 营造层级 |
| **像素感与科技感并存** | 预览区保留 pixelated 渲染，呼应游戏素材本质 |

## 2. 色彩系统

### 2.1 中性色阶（背景与文字主体）

| Token | 值 | 用途 |
|-------|-----|------|
| `--bg-0` | `#08090d` | 最底层画布 |
| `--bg-1` | `#0d0f15` | 面板背景 |
| `--bg-2` | `#12151d` | 卡片/顶栏 |
| `--bg-3` | `#171b25` | 悬浮/输入态 |
| `--line` | `#222734` | 主分隔线 |
| `--txt-0` | `#f2f4f8` | 主文字 |
| `--txt-1` | `#aab2c5` | 次级文字 |
| `--txt-2` | `#6c7488` | 弱化文字/标签 |
| `--txt-3` | `#454c5e` | 提示/占位 |

### 2.2 功能强调色（霓虹）

| Token | 值 | 语义 |
|-------|-----|------|
| `--acc`（蓝） | `#5b8cff` | 主强调、激活、运行 |
| `--cyan` | `#3ee0d0` | 进度、数据流 |
| `--violet` | `#a571ff` | 编排类节点 |
| `--pink` | `#ff6ba6` | 处理类节点 |
| `--amber` | `#ffb648` | 像素类节点、警示 |
| `--green` | `#56d97a` | 成功、在线、缓存命中 |

> 节点按类别配色：生成=蓝 / 处理=粉 / 像素=琥珀 / 编排=紫 / 导出=绿。状态灯一律用「色 + 同色 glow 阴影」表现发光。

## 3. 字体系统

| 用途 | 字体 | 说明 |
|------|------|------|
| 界面正文/标题 | SF Pro Display / system-ui | 字重 400/500/600 三档 |
| 数值/代码/标签 | SF Mono / JetBrains Mono | 参数值、能力名、hash 一律等宽 |

字号档位：标题 14-15px / 正文 12-13px / 标签 10-11px / 微标 8.5-9px。
字距：正文 `+0.1px`，大写小标题 `+1px~1.2px`。

---

## 4. 形状与空间

| Token | 值 | 用途 |
|-------|-----|------|
| 圆角 小 | `8px` | 按钮、输入、节点头像 |
| 圆角 中 | `12px` | 节点卡片 |
| 圆角 大 | `18px` | 大面板/容器 |
| 状态灯/端口 | 圆形 | 6-11px |

间距用 4 的倍数（4/8/12/16）。节点宽度统一 172px，保证画布整齐。

## 5. 关键界面布局

三栏式专业工具布局（顶栏 + 左中右 + 状态栏）：

```
┌──────────────── 顶栏：Logo · 面包屑 · 引擎状态 · 运行按钮 ────────────────┐
├──────────┬────────────────────────────────────┬──────────────────────┤
│ 左：节点库 │ 中：节点画布（网格背景+玻璃节点+霓虹连线）│ 右：检查器（参数/路由/血缘）│
│ 分类+搜索 │ 缩放/平移/添加节点                    │ 选中节点的全部配置        │
├──────────┴────────────────────────────────────┴──────────────────────┤
└──────────────── 状态栏：引擎/节点数/缓存/进度条 ──────────────────────────┘
```

### 关键视觉元素

- **画布网格**：26px 网格 + 径向遮罩（中心清晰、边缘淡出），底部叠加蓝/紫辉光团
- **节点卡片**：玻璃态（半透明 + blur）+ 顶部彩色状态点 + 右上角能力标签 + 预览缩略区
- **连线**：贝塞尔曲线，蓝→紫渐变；激活端口带 glow
- **状态灯**：所有「在线/成功/缓存」状态用发光圆点表达
- **进度条**：蓝→青渐变 + 呼吸动画

## 6. 动效原则

| 场景 | 动效 |
|------|------|
| 悬浮 | 边框提亮 + 阴影加深，0.14-0.18s |
| 运行按钮悬浮 | 上移 1px + 阴影扩散 |
| 进度条 | 呼吸 opacity 2s 循环 |
| 节点连线 | （可选）数据流光点沿线移动 |

克制为主——专业工具的动效是反馈，不是表演。

---

## 7. 落地建议

- 前端用 React + React Flow，把上述 token 写成 CSS variables 或 Tailwind config
- 节点画布直接用 React Flow 的自定义节点（custom node）渲染玻璃卡片
- 暗色为默认主题；如需亮色主题，反转中性色阶、强调色降饱和即可
- 预览区图片务必 `image-rendering: pixelated`，保住像素质感
