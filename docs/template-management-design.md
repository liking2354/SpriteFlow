# 模板管理前端设计

## 一、页面结构

```
/specs     → 规格书管理页（主入口）
/specs/:id → 规格书编辑器（图层/分块/Prompt 预览）
/characters → 角色模板库
/actions   → 动作模板库
/vfx       → 技能特效模板库
/pipelines → 阶段管线编辑器
/batch     → 批量生成页面（选Spec+角色+动作→一键生成）
```

侧边导航新增 "模板管理" section，包含上述子菜单。

## 四、关键技术点

### 4.1 Prompt 预览：实时但不调用 API

前端可以直接实现拼接逻辑（减少一次网络请求）：
```typescript
// 前端也可以调 /api/templates/preview 获得服务端权威结果
const preview = useQuery({
  queryKey: ["template-preview", specId, charId, actionId],
  queryFn: () => api.previewPrompt({ spec_id, character_template_id, action_template_id }),
  enabled: !!specId,
});
```

### 4.2 图层拖拽排序

使用 `@dnd-kit/core` 实现拖拽，调整 `sort_order` 后保存。

### 4.3 中间结果实时预览

测试变更时：
1. 改一个 Block 内容 → 保存
2. 用该 Spec + 固定角色/动作 → 快速生成 1 张图（走 /api/generate 轻薄模式）
3. 在新 Tab 对比新旧结果

### 4.4 模板版本控制

每次编辑 Spec 时 `version += 1`，保留历史版本以便回滚。

## 五、总结

| 模块 | 作用 | 频率 |
|------|------|------|
| SpriteSpec 编辑器 | 管理画布规格+Prompt图层 | 偶尔（调好后很少变）|
| Character 模板库 | 管理角色定义 | 新增角色时 |
| Action 模板库 | 管理动作Prompt | 新增动作类型时 |
| VFX 模板库 | 管理技能特效 | 新增技能时 |
| Pipeline 编辑器 | 编排阶段流程 | 设计新管线时 |
| Batch 批量生成 | 选Spec+角色+动作→批量产出 | **最高频使用** |
